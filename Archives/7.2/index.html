Soar Version 7.2 Release

This is the working page for the Soar 7.2 Release(9/1/98). Release Notes are under construction and will be available shortly. Soar 7.2 requires Tcl7.6 and Tk4.2 on all systems (Mac, Windows and Unix). If you are already running Eaters, TankSoar, or the AAAI-97 version of Soar (Soar 7.1), you already have Tcl7.6 and Tk4.2. If you are upgrading from Soar 7.0.*, or don't have Soar at all yet, you will have to check to see if you have Tcl, and if so, what version you have. Soar 7.2 is also compatible with Tcl 8.0, but it does requires that you rebuild Soar, linking with Tcl/Tk8.0. The binaries provided below were built for Tcl 7.6/Tk4.2. The README.(systype) files in the distribution will tell you how to check for Tcl on your system, and what to do if you don't have it.

Soar 7.2 for Windows

  • Soar72.EXE. (2.7 Mb) The self-installing binaries for Windows 95 / Windows NT (and possibly even Windows 3.1 with Win32s installed, but we haven't tested that).

    The standard installation location for Tcl/Tk on Windows is in "C:\Program Files\TCL" If you don't see Tcl there, first do a File Find on your system to see if someone installed Tcl somewhere else. If you don't have Tcl7.6/Tk4.2 already installed on your system, grab the Windows 3.1, Windows 95, and Windows NT self-extracting installer (tcl7_6/win76p2.exe): (about 1.5 Mbytes) The file is a self-extracting executable. It will install the Tcl and Tk libraries, the wish and tclsh programs, and documentation.

    THEN, because we think these Tcl/Tk binaries are slightly buggy, we recommend that you apply the following patches. that Chris Waterson developed, using the README file he wrote. This should clear up any segfaults that appear on some systems when exiting Soar. If Tcl/Tk was previously installed on your system and you get the seg faults when you exit Soar, you should install the patches. If you choose not to install these patches, you will probably get warning messages from the Windows operating system whenever you exit Soar. They are usually pretty harmless, although some people have reported that they have occasionally had to reboot after these errors.

    If at all possible, use the defaults for file locations and installation settings. Soar-7.2 will require no reconfiguration for your Windows system if you have installed Tcl/Tk on the C: drive (specifically in "C:\Program Files\TCL", and if Soar is unzipped to c:\ (it will default to a folder called Soar-7.2). Just open the Soar-7.2 folder and dbl-click on the Soar icon (a feather, for now) to start Soar. If you need to use locations other than the defaults, or if you aren't able to run Soar by dbl-clicking on the Soar icon, see the README.Win file for information about what you will need to change to run Soar.

    Soar 7.2 for Macintosh

  • Soar72.sea.hqx. (6.9 Mb) A BinHex 4.0 self-extracting archive that includes the binary distributions for Power PC Macs. This archive also includes the Tcl/Tk Installer for installing Tcl7.6/Tk4.2. (If you already have this version of Tcl/Tk installed, you can just dump the Installer program in the trash.) Download and extract the Soar archive file (by dbl-clicking on the Soar-72.sea icon), which will create a folder called Soar-7.2. See the README.Macintosh file to get started.

    Soar 7.2 for Unix

  • Soar72.tgz. (2.1 Mb) A gnu-zipped tar file which includes the precompiled libraries for Linux. See the README.Unix file for information on running Soar and on building Soar for other Unix systems. If there is enough demand, we can possibly build and post libraries for SunOS 4.1.3, Solaris 5.5.1, Linux 2.0.30, and IRIX 5.3. If you need to install Tcl7.6 and Tk4.2, you can obtain the code and follow the instructions found on the Scriptics Tcl7.6 Download Page. About a third of the way down that page you'll find the info for Unix sources.
